# Observations

A living document for capturing issues, rough edges, and improvement ideas as they're noticed during normal development. Updated organically -- not investigated unless explicitly instructed.

## Rules

- **Add** an entry whenever you notice something during any task. One line is enough.
- **Behavioral only**: log things that affect users, cause bugs, or create friction. Skip structural commentary (file too long, naming style) unless it directly caused a problem.
- **Skip duplicates**: don't log anything already tracked in `docs/reference/issues/` or the agent guide's known tech debt section.
- **Don't investigate** unless the user explicitly asks. Just record and move on.
- **Check off** `[x]` when an observation is resolved. Add the fix date.
- **Prune** checked items when the list gets long or when instructed.
- **Promote** to `docs/reference/issues/` if an observation needs formal tracking with code evidence and severity.
